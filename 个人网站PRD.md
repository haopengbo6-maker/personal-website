# 郝鹏博个人网站 PRD

## 1. 项目概述

### 1.1 项目名称

郝鹏博个人网站

### 1.2 项目目标

构建一个可长期维护、可部署到 GitHub Pages 的个人网站，用于展示个人身份、教育背景、项目能力、技术栈、校园经历、兴趣爱好与联系方式。

网站不只是“线上简历”，而是一个具有个人气质的数字名片。它需要同时满足三个目标：

- 让老师、同学、招聘方快速了解个人背景与技术能力。
- 让访问者看到项目实践能力，尤其是 AI、数据分析、前端交互和游戏 AI 相关经历。
- 展示个人兴趣与审美，让网站具有记忆点和人格化表达。

### 1.3 核心定位

个人定位建议：

> AI student / Builder / Visual thinker

中文表达：

> 我在学习人工智能，也在用代码、图像和交互表达自己的观察。

关键词：

- 北京航空航天大学人工智能本科
- AI 项目实践者
- 数据分析与交互工具开发
- 视觉创作与摄影爱好者
- 游戏机制与用户体验观察者

## 2. 目标用户

### 2.1 主要访问者

- 实习招聘方：快速判断技术能力、项目经历和成长潜力。
- 老师与学长学姐：了解学习背景、项目方向和合作可能。
- 同学与朋友：了解个人兴趣、作品和社交联系方式。
- 潜在合作伙伴：查看项目、创作方向和个人风格。

### 2.2 用户核心诉求

- 你是谁？
- 你学什么、会什么？
- 你做过哪些项目？
- 你的项目是否真实、清晰、有思考？
- 你除了技术之外还有什么特点？
- 如何联系你？

## 3. 设计方向

### 3.1 整体风格

推荐风格：暗色电影感 + 理工极简 + 轻微艺术实验感。

原因：

- 用户提供的个人照片为黑色背景、强光影、正式肖像，非常适合作为首页主视觉。
- 简历内容偏 AI、数据、项目实践，适合冷静、清晰、可信的视觉系统。
- 兴趣爱好包含美术、摄影、音乐、游戏、美食，适合在局部页面加入更强的动态表达。

### 3.2 视觉关键词

- 黑色背景
- 低饱和冷色强调
- 细线网格
- 光影流动
- 清晰排版
- 克制动效
- 项目卡片明确
- 局部页面有独特交互

### 3.3 色彩建议

主色：

- 背景黑：`#050505`
- 深灰背景：`#101114`
- 主文字：`#F4F4F5`
- 次级文字：`#A1A1AA`
- 边框：`rgba(255,255,255,0.12)`

强调色：

- 冷蓝：`#7DD3FC`
- 银白：`#E5E7EB`

使用原则：

- 全站只保留一个主强调色，避免多色混乱。
- 不使用大面积 AI 紫色渐变。
- 不使用过度玻璃拟态。
- 项目卡片以结构清晰为主，动效服务于信息表达。

### 3.4 字体建议

中文：

- Microsoft YaHei
- Noto Sans SC
- Source Han Sans SC

英文与代码：

- Geist
- Satoshi
- JetBrains Mono

## 4. 技术方案

### 4.1 技术栈

- 前端框架：React
- 构建工具：Vite
- 样式方案：CSS Modules / 普通 CSS / Tailwind CSS 三选一
- 路由：React Router
- 动效：Framer Motion 或 CSS animation
- 部署：GitHub Pages

### 4.3 可编辑性要求

为了方便后续维护，网站内容应尽量放在 `src/data/` 中。

示例：

- 项目内容放在 `projects.js`
- 技能内容放在 `skills.js`
- 兴趣爱好放在 `interests.js`
- 联系方式放在单独配置对象中

后续修改内容时，尽量不需要改组件结构。

## 8. 交互与动效要求

### 8.1 全局交互

- 导航栏固定或半固定。
- 点击导航平滑滚动或跳转页面。
- 项目卡片 hover 时轻微上浮。
- 按钮有明确 hover 和 active 状态。
- 页面切换自然，不突然闪烁。

### 8.2 动效原则

- 动效服务于信息表达，不做无意义炫技。
- 首页动效克制、高级、慢速。
- 兴趣页面可以更有个性，但仍保持统一视觉系统。
- 移动端减少复杂动效。
- 支持 `prefers-reduced-motion`，尊重减少动画的系统设置。

## 9. 响应式要求

### 9.1 桌面端

- 最大版心控制在 1700px 左右。
- 内容区根据模块设置合理宽度。
- Hero 可左右分栏。
- 项目卡片可三列或不规则网格。

### 9.2 平板端

- Hero 可变为上下结构。
- 项目卡片变为两列。
- 兴趣入口保持清晰点击区域。

### 9.3 手机端

- 单列布局。
- 导航折叠为菜单。
- 图片不遮挡文字。
- 按钮和卡片触控区域足够大。
- 避免横向滚动。
